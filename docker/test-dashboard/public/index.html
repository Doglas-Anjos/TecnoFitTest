Race Condition Tests

These tests verify that the API correctly handles concurrent requests and prevents double spending.

1. Double Withdrawal Attack

Two simultaneous requests try to withdraw R$ 600 from an account with R$ 1000. Only one should succeed.

# Account: 00000000-0000-0000-0000-000000000001 (R$ 1000) # 2 concurrent requests for R$ 600 each curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000001/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"test1@test.com"},"amount":600,"schedule":null}' # Expected: 1 success (200), 1 locked (423) or insufficient (422)

2. Balance Overflow

10 simultaneous requests try to withdraw R$ 10 from R$ 50 balance. Maximum 5 should succeed.

# Account: 00000000-0000-0000-0000-000000000003 (R$ 50) # 10 concurrent requests for R$ 10 each curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000003/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"test3@test.com"},"amount":10,"schedule":null}' # Expected: Max 5 success, rest locked/insufficient

3. Scheduled + Immediate

One scheduled and one immediate withdrawal at the same time. Immediate should execute, scheduled should be queued.

# Account: 00000000-0000-0000-0000-000000000002 (R$ 500) # 2 concurrent: 1 immediate + 1 scheduled # Immediate (R$ 300) curl -X POST .../withdraw -d '{"amount":300,"schedule":null}' # Scheduled (R$ 300) curl -X POST .../withdraw -d '{"amount":300,"schedule":"2025-12-31 23:59:59"}'

4. Zero Balance Withdrawal

Attempt to withdraw from an account with zero balance. Should return 422 error.

# Account: 00000000-0000-0000-0000-000000000004 (R$ 0) curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000004/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"test4@test.com"},"amount":1,"schedule":null}' # Expected: 422 Insufficient Balance

5. Locked Account

Attempt to withdraw from a locked account. Should return 423 error.

# Account: 00000000-0000-0000-0000-000000000006 (Locked) curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000006/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"test6@test.com"},"amount":100,"schedule":null}' # Expected: 423 Account Locked

6. Invalid Amounts

Attempt to withdraw negative or zero amounts. Should return 422 validation error.

# Test negative and zero amounts curl -X POST .../withdraw -d '{"amount":-100,...}' # Expected: 422 Validation Error curl -X POST .../withdraw -d '{"amount":0,...}' # Expected: 422 Validation Error

7. Negative Balance Account

Attempt to withdraw from an account with negative balance. Should return 422 error.

# Account: 00000000-0000-0000-0000-000000000005 (R$ -100) curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000005/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"test5@test.com"},"amount":1,"schedule":null}' # Expected: 422 Insufficient Balance

8. Invalid PIX Key

Attempt to withdraw using a PIX key that doesn't belong to the account. Should return 422 error.

# Account: 00000000-0000-0000-0000-000000000001 # Using wrong PIX key (not registered to this account) curl -X POST http://localhost:9502/account/00000000-0000-0000-0000-000000000001/balance/withdraw \ -H "Content-Type: application/json" \ -d '{"method":"PIX","pix":{"type":"email","key":"wrong@email.com"},"amount":100,"schedule":null}' # Expected: 422 Invalid PIX Key
